# Openbulls Architecture

## High-level

Openbulls is a TypeScript monorepo. The **central runtime is LangGraph**;
long-running agent workflows are queued and executed by `apps/agent-worker`,
not by `apps/web`.

## Architectural rules

1. **Apps are thin** вҖ” business logic lives in packages.
2. **`apps/web` does not execute long workflows** вҖ” it enqueues them.
3. **`apps/agent-worker` runs queued work** вҖ” LangGraph runs, scheduled jobs,
   report generation, notifications.
4. **`apps/cron` is generic** вҖ” finds due `user_scheduled_jobs` and enqueues
   them; never hard-codes a job type.
5. **Packages own business logic** вҖ” `domain / application / infrastructure`.
6. **Market data is its own package** вҖ” TCMB, KAP, Yahoo, TwelveData, BIST.
7. **`packages/integrations` is glue** вҖ” generic external utilities only.
8. **AI tools call package services** вҖ” no business logic inside tools.
9. **`packages/agent-runtime` owns graph orchestration** вҖ” no graph code in
   apps.
10. **Parallel agents are graph branches** вҖ” `Send` / subgraph pattern, not
    `Promise.all` scattered in services.
11. **Billing is central** вҖ” `reserve-credit` / `finalize-usage` / `refund`.
12. **Agent runs are resumable** вҖ” checkpointed in DB, paused on credit loss.
13. **Strict TypeScript** вҖ” zod at boundaries, branded IDs, no `any`.
14. **Drizzle + repository ports** вҖ” domain packages depend on ports, not Drizzle.
15. **Next.js App Router + shadcn/ui + ai-elements** вҖ” server actions in
    `apps/web/features/`.
16. **Package build via tsup** вҖ” `package.json / tsconfig.json / tsup.config.ts`.
17. **Naming**: kebab-case files, PascalCase classes, camelCase functions.
18. **Code style**: include full file paths and complete contents.
19. **Common commands**: see `README.md`.
20. **LangChain deps** in `packages/agent-runtime`.
21. **Current focus**: clean boundaries, LangGraph-centered runtime, billing
    safety, market-data separation.
